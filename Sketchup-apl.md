如果已经 add 了文件，又改了内容，只需**重新执行 git add 文件名** —— Git 会用「工作区的最新版本」覆盖暂存区的旧版本：

bash

运行

```bash
# 补救：重新add，让暂存区同步最新编辑的版本
git add test.txt

# 再提交，此时提交的就是最新版本了
git commit -m "feat: 完善test.txt内容"
```

### 四、总结

| 操作顺序             | 结果                           | 适用场景                 |
| -------------------- | ------------------------------ | ------------------------ |
| 先编辑 → 后 add     | 暂存区 = 最新版本，提交正确    | 99% 的常规场景（推荐）   |
| 先 add → 后编辑     | 暂存区 = 旧版本，提交错误      | 无合理场景，属于操作失误 |
| add 后编辑 → 重 add | 暂存区更新为最新版本，提交正确 | 补救操作（偶尔用到）     |

核心原则：`git add` 是「把当前工作区的文件状态存入暂存区」的动作，一定要等文件内容编辑到你想要提交的状态后，再执行 `add`。
